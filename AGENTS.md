# AGENTS.md

## 目的

このプロジェクトは Django アプリである。
Codex は、主に静的ファイルの CSS を作成・修正し、画面全体の見た目と使いやすさを向上させることを目的とする。

単にCSSを当てるだけでなく、Djangoテンプレート上で自然に使える、保守しやすく、完成度の高いUIを目指す。

## 最優先方針

* 既存の機能を壊さない
* Djangoの構成に合った形でCSSを作成する
* HTMLテンプレートの構造を確認してからCSSを書く
* 見た目のクオリティを重視する
* ただし、過度に複雑・派手なデザインにはしない
* 大学生の課題・個人開発として自然で、清潔感のあるUIにする

## 作業範囲

主に以下を行う。

* `static/` 配下のCSSファイル作成・編集
* Djangoテンプレートに合ったclass名の整理
* 必要に応じたテンプレート側の軽微な修正
* レイアウト、余白、配色、フォント、ボタン、フォーム、カード、一覧画面の調整
* スマホ表示を考慮したレスポンシブ対応

以下は原則として行わない。

* views.py の処理変更
* models.py の変更
* urls.py の変更
* JavaScriptの追加
* 外部CSSライブラリの追加
* 既存機能を変えるような大きなHTML構造変更

ただし、CSSを適切に適用するために必要な `class` の追加や、テンプレートの軽微な整理は行ってよい。

## Djangoでの注意点

* テンプレートでCSSを読み込む場合は、Djangoのstatic機能を使うこと
* テンプレートの先頭に必要であれば以下を追加すること

```django
{% load static %}
```

* CSSファイルの読み込みは、基本的に以下のような形にすること

```django
<link rel="stylesheet" href="{% static 'css/style.css' %}">
```

* 既存のstaticファイル構成がある場合は、それに合わせること
* 新しくCSSファイルを作る場合は、基本的に以下を想定する

```text
static/
  css/
    style.css
```

* アプリごとにstaticを分けている構成の場合は、既存の構成を優先すること

## デザイン方針

全体として、以下のような雰囲気を目指す。

* シンプル
* 清潔感がある
* 余白がきれい
* 文字が読みやすい
* 操作しやすい
* 画面に統一感がある
* 課題提出でも違和感のない完成度
* 個人開発としても見栄えがするUI

派手すぎるアニメーションや装飾は避ける。
見た目だけでなく、使いやすさを重視する。

## CSSの書き方

* classセレクタを中心に使う
* idセレクタへの依存はなるべく避ける
* `!important` は原則使わない
* 同じ指定を何度も書かない
* 余白、色、角丸、影、文字サイズに一貫性を持たせる
* コメントは必要最低限にする
* 初心者でも理解しやすいCSS構成にする
* 長くなりすぎる場合は、セクションごとにコメントで整理する

例：

```css
/* Base */
body {
}

/* Layout */
.container {
}

/* Components */
.card {
}

/* Forms */
.form-group {
}

/* Responsive */
@media (max-width: 768px) {
}
```

## レイアウト方針

* 画面中央に自然に収まるレイアウトにする
* 最大幅を設定し、横に広がりすぎないようにする
* セクションごとの余白を適切に取る
* ボタンやリンクの位置を分かりやすくする
* 一覧画面では情報のまとまりが分かるようにする
* フォーム画面では入力欄を見やすく、操作しやすくする

## フォームの方針

Djangoアプリではフォーム画面の見た目が重要なので、以下を意識する。

* label と input の対応が分かりやすい
* 入力欄の幅を自然にそろえる
* ボタンは押せることが直感的に分かるデザインにする
* エラー表示がある場合は目立つが不自然でないデザインにする
* 送信ボタン、キャンセルリンク、戻るリンクの区別を分かりやすくする

## ボタンの方針

ボタン・リンクボタンは用途ごとに見た目を整理する。

例：

* 主要操作: 登録、保存、追加
* 通常操作: 詳細、編集、戻る
* 危険操作: 削除

危険操作のボタンは、他のボタンと見分けがつくようにする。

## レスポンシブ対応

スマートフォン表示でも崩れないようにする。

特に以下を守る。

* 横スクロールを出さない
* 画面幅が狭いときは要素を縦に並べる
* フォームの入力欄を画面幅に合わせる
* ボタンをタップしやすい大きさにする
* カードやテーブルが画面外にはみ出さないようにする
* 必要に応じて `@media (max-width: 768px)` を使う

## 品質基準

作業後の画面が以下を満たすようにする。

* パッと見て未完成感がない
* 文字が読みやすい
* 余白が詰まりすぎていない
* 色数が多すぎない
* ボタンの役割が分かりやすい
* フォームが使いやすい
* PCでもスマホでも大きく崩れない
* CSSが読みやすく整理されている

## 変更時の注意

* 既存テンプレートを確認してから変更する
* 既存classがある場合はできるだけ活用する
* 必要な場合のみclassを追加する
* 意味のないdivやclassを大量に増やさない
* CSSのためだけにHTMLを不自然にしない
* 既存機能に影響する変更は避ける
* 変更前後で主要画面を確認する

## 禁止事項

* 既存機能を壊すこと
* Djangoのviews, models, urlsを理由なく変更すること
* 外部ライブラリを勝手に追加すること
* JavaScriptを勝手に追加すること
* `!important` を多用すること
* テンプレート構造を大きく作り替えること
* 使われていないCSSを大量に追加すること
* 過度に派手な装飾をすること

## 作業手順

1. プロジェクト構成を確認する
2. Djangoテンプレートの場所を確認する
3. staticファイルの構成を確認する
4. 既存HTMLのclass名・id名を確認する
5. CSSを作成または修正する
6. 必要に応じてテンプレートにclassを追加する
7. PC表示とスマホ表示を考慮する
8. 変更内容を簡潔に説明する

## 作業後の説明

作業後は、以下を簡潔に説明する。

* 変更したファイル
* 追加・修正した主なデザイン
* どの画面を整えたか
* レスポンシブ対応の有無
* 注意点や未確認事項
